Dataset Overview¶

This data set includes information about individual rides made in a bike-sharing system covering the greater San Francisco Bay area.

The dataset contains 15 features of 171,230 bike trips. The included features includes:

  • Trip Infomation: duration(min),start_hour ,end_hour ,start_day, end_day, day_of_month, week_in_month,bike_id, and distance(km)

  • Station Information: start_station_name and end_station_name

  • Member Infomation: user_type, member_gender, bike_share_for_all_trip and member_age

(Hourly Bike Usage)¶

It is observed that 8am and 5pm are the hour with the most activties indicating most of the trips are immediately before working hours 7am , in between working hours 10am to 3pm, and immediately after working hours 6pm to 7pm.

(How Each User Type Uses The Services Hourly )¶

We have seen how both users combined uses the service hourly but now we want to look at how both Subscribers and Customers uses it.

Generally, Subscribers have the higher number of trips.

  • Start hour 8am and 5pm is the peak hour for both customers and subscribers

  • End hour 8am and 5pm is the peak hour for Subscribers while 5pm and 6pm is the peak hour for Customers.

( How Each User Type Uses The Services Daily )¶

Generally, Subscribers have a higher number of trips per day. Trips on weekdays(Mon- Fri) have a higher number of trips compared to the weekends(Sat-Sun) as expected. Although this only applies to Subscribers. With Customers having a fairly constant number of trips in a day
